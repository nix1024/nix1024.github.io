ARC 简介

从 iOS 5开始，OC 引进了一种 ARC（Automatic Reference Counting）的方式来管理内存，由编译器来为我们生成相应的内存管理代码。

在类属性的声明中，ARC 相关的修饰词有两个：

  • strong 一个对象只要存在一个 strong 类型的指针指向它，就不会被释放。
  • weak 不会延长对象的生命周期，当weak类型的指针指向的对象被释放时，指针自身会被自动置为nil。

在 ARC 机制下，指针缺省的类型是strong。

在声明局部变量时，有四个 ARC 相关的修饰词，他们的用法跟 const 类似。

  • __strong 缺省值。用法同strong。
  • __weak 用法同weak。
  • __unsafe_unretained 类似于weak，但其指向的对象被释放时，其值不会被置为nil。应该是由于兼容性的原因而存在，通常不建议使用。
  • __autoreleasing 用来修饰一个以引用（id *）方式传入的参数，当函数返回值时被释放。

上一次更新博客已经是4月份了。我还在。

为了证明这个博客存在的意义并不只是为了找工作时给面试官一个好一些的印象，我决定继续更新下去。

一觉睡到10点多，起来整了整博客主题，去掉了侧边栏，看起来更简单一些。但还是没有考虑字体的选择，看了看专业前端的作品，自叹不如。为了对得起 farbox 官网上的链接，就没有切到自带的主题。

farbox 在我不在的这段日子里有了长足的进步，后台界面设计更加精致，自带的 editor 和 code 简直是神器！还有那个自动同步功能是怎么做到的！

下午打了几把 dota 睡了一觉就过去了……╮(╯_╰)╭

最近从零开始学习 iOS 开发，深深地感觉到封闭系统带来的易用性，甚至连开发也是如此，Storyboard 用得飞起。系统适配也非常方便，模拟器可以模拟所有型号的 iPhone，安卓开发没有接触过，想想应该是很痛苦的。

跟我一同学习的 Sunus 同学对此表示不屑，坚持用更 Geek 的方式——纯代码来写界面。我喜欢 IB 的易用和效率，他喜欢代码的可读性和可维护性。

本来看到 App Store 上的汇率转换 App 都不是很好用，准备写一个简单的练练手，直到发现一款极简汇率，无论是设计、交互以及流畅性都堪称一流，这么好的 App 居然还是免费的！看了下作者的博客，很巧的是，他的博客也托管在 farbox 上。他还翻译了苹果的 HID，果断下载学习。

接下来准备在空闲时间里做一些小的 Demo 练手，在做的过程中学习永远是最快的。
